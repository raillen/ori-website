Ori makes control flow explicit.

Absence uses `optional<T>`.

Failure uses `result<T, E>`.

Branching over data uses `match`.

## Structs

Use a `struct` for data with named fields.

```ori
struct Book
    title: string
    author: string
    year: int
end

const book: Book = Book(
    title: "Clean Code",
    author: "Robert C. Martin",
    year: 2008,
)
```

Fields are named at construction. This keeps call sites readable.

### Field contracts

Fields can declare a validation rule with `if it`:

```ori
struct Rectangle
    width: int  if it > 0
    height: int if it > 0
end
```

`it` refers to the field value being validated. Contracts are checked at construction time and on mutation. A violation is a runtime panic.

### Anonymous struct form

When the expected type is already visible, use the anonymous struct form:

```ori
const short_book: Book = .{
    title: "Clean Code",
    author: "Robert C. Martin",
    year: 2008,
}
```

Use the full `Book(...)` form when there is no expected type nearby.

### Struct update

To copy a struct with a few changed fields, use `with`:

```ori
const newer_book: Book = book with {
    year: 2009,
} end
```

`with` creates a new value. It does not mutate the original binding.

## Enums and `match`

Use an `enum` for state that has a fixed set of cases.

```ori
enum Command
    Add(title: string)
    List
    Quit
end
```

Handle the state with `match`:

```ori
match command
    case Command.Add(title):
        io.print(f"adding {title}")
    case Command.List:
        io.print("listing")
    case Command.Quit:
        io.print("bye")
end
```

`match` must be exhaustive: every possible value must be covered.
Use `case else` as the explicit fallback for non-exhaustive cases.

```ori
match command
    case Command.Add(title):
        io.print(f"adding {title}")
    case else:
        io.print("unknown command")
end
```

For domain states, prefer naming every case.

When the expected enum type is visible, the enum name can be omitted:

```ori
const command: Command = .List
const add: Command = .Add(title: "Clean Code")
```

Payload variants use named arguments inside `(...)` in the current compiler.

### Guards

Guards help when a case needs an extra condition:

```ori
match score
    case n if n >= 90:
        io.print("A")
    case n if n >= 80:
        io.print("B")
    case else:
        io.print("C")
end
```

Guarded cases do not count toward exhaustiveness.

## `optional<T>`

Use `optional<T>` when a value may not exist.

```ori
func find_title(id: int) -> optional<string>
    if id == 1
        return some("Ori Guide")
    end
    return none
end
```

Read it with `match`:

```ori
match find_title(1)
    case some(title):
        io.print(title)
    case none:
        io.print("not found")
end
```

Or bind the happy path with `if some`:

```ori
if some(title) = find_title(1)
    io.print(title)
end
```

## `result<T, E>`

Use `result<T, E>` when an operation can fail with a reason.

```ori
func divide(a: int, b: int) -> result<int, string>
    if b == 0
        return error("division by zero")
    end
    return success(a / b)
end
```

Use `try` to propagate failure from a function that also returns `result`.

```ori
func double_division(a: int, b: int) -> result<int, string>
    const value: int = try divide(a, b)
    return success(value * 2)
end
```

Use `match` at the boundary where the program can explain the result to a person.

## Loops

Use `for` when iterating values:

```ori
const names: list<string> = ["Ada", "Grace"]

for name in names
    io.print(name)
end
```

Use `while` when the loop depends on a condition:

```ori
var index: int = 0
while index < len(names)
    io.print(names[index])
    index = index + 1
end
```

Use `loop` with `break` when the exit point is inside the body.

## Closures and pipes

Use `do` for small anonymous functions.

```ori
import ori.iter as iter

const numbers: list<int> = [1, 2, 3, 4]

const even: list<int> =
    numbers
    |> iter.filter(do(n: int) => n % 2 == 0)
```

Prefer named functions when the rule has a domain name.

```ori
func is_even(n: int) -> bool
    return n % 2 == 0
end

const even: list<int> = iter.filter(numbers, is_even)
```

## Collection literals

```ori
const names: list<string> = ["Ada", "Grace"]
const scores: map<string, int> = {"Ada": 10, "Grace": 9}
const tags: set<string> = set{"compiler", "docs"}
const pair: tuple<int, string> = tuple(1, "one")
```

Empty collections need a type annotation:

```ori
const empty_names: list<string> = []
const empty_scores: map<string, int> = {}
```

## Ranges and slices

Ranges are inclusive and can be descending:

```ori
for index in 0..3
    io.print(string(index)) -- 0, 1, 2, 3
end

for index in 3..0
    io.print(string(index)) -- 3, 2, 1, 0
end
```

Float ranges are not part of the current language. Use an explicit loop or iterator helper when a floating-point step is needed.

Slices use an exclusive end:

```ori
const middle: list<string> = names[1..3]
const tail: list<string> = names[1..]
const head: list<string> = names[..3]
const copy: list<string> = names[..]
```

## Clean flow checklist

- Use `optional<T>` for missing values.
- Use `result<T, E>` for failures with a reason.
- Use `match` when the reader should see every case.
- Use `try` only inside a compatible return type.
- Prefer `for` over manual indexes when the index is not important.
- Keep closures short; name longer rules.
