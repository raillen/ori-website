## Why Ori?

In a landscape where many languages favor shortcuts and implicit behavior, Ori takes a different path. Code must be written to be understood.

<CardGrid>
  <Card title="Readability over brevity" icon="open-book">
    Blocks use `end` instead of braces. No cryptic symbols. Every decision is visible in the source code itself.
  </Card>
  <Card title="Explicit types everywhere" icon="approve-check-circle">
    Type annotations are required at declarations. No hidden coercion, no implicit conversions — what you see is what runs.
  </Card>
  <Card title="Zero-toolchain linker" icon="setting">
    The compiler handles CRT discovery and linking automatically. No C or Rust toolchain needed on the user machine.
  </Card>
  <Card title="JIT and AOT compilation" icon="rocket">
    Run code instantly in-memory with Cranelift JIT, or compile to standalone native binaries for distribution.
  </Card>
</CardGrid>
