## The Impulse Behind Ori

Ori was born out of a specific frustration: **the readability crisis in modern software**. As codebases grow, they spend 90% of their lifecycle being read by humans, yet modern languages introduce excessive visual noise, magic implicit casts, and unpredictable control flows.

Ori is an explicit, lightweight language that targets machine code directly via Cranelift. It introduces a clean, keyword-delimited syntax designed to keep codebases readable, type-safe, and self-documenting.

## Language Pillars

<CardGrid>
  <Card title="Readability First" icon="open-book">
    No cryptic brackets or symbol noise. Blocks are structured using clear text delimiters like `end`.
  </Card>
  <Card title="Strict Explicit Types" icon="approve-check-circle">
    Required annotations at declarations. Type coercion is entirely explicit via primitive functions (e.g., `int(val)`).
  </Card>
  <Card title="Zero-Toolchain Linker" icon="setting">
    The compiler automatically orchestrates CRT discovery and linking via bundled `rust-lld` or system linkers.
  </Card>
  <Card title="JIT and AOT Compilation" icon="rocket">
    AOT compiles to standalone binaries. JIT executes instantly in-process via Cranelift, resolving symbols dynamically.
  </Card>
</CardGrid>

## Quick showcase

An Ori source file starts with a namespace declaration, imports dependencies, and uses clear, readable control flows:

```ori
namespace app.main

import ori.io as io

func main()
    const label: string = if true then "Ori" else "world"
    io.println("Hello, " + label + "!")
end
```

To run this file instantly with in-memory JIT execution:

```bash
ori run main.orl
```
