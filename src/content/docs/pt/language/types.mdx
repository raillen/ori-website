A linguagem Ori é **explicitamente tipada**: todas as variáveis exigem anotações de tipo no momento da declaração.

## Primitivos

| Tipo | Exemplo |
|------|---------|
| `bool` | `true`, `false` |
| `int` | `42` |
| `int8`, `int16`, `int32`, `int64` | `12i8`, `12i16`, `12i32`, `12i64` |
| `u8`, `u16`, `u32`, `u64` | `12u8`, `12u16`, `12u32`, `12u64` |
| `float` | `3.14` |
| `float32`, `float64` | `3.14f32`, `3.14f64` |
| `string` | `"olá"` |
| `bytes` | Buffers de bytes brutos |
| `void` | Sem valor (apenas para instruções) |

## Sufixos Numéricos

A largura de um literal numérico é escolhida com um sufixo no literal. O sufixo
não é o nome do tipo.

```ori
const default_int: int = 12
const small: int8 = 12i8
const byte: u8 = 255u8
const compact_float: float32 = 3.14f32
```

- **Sufixos de inteiros com sinal**: `i8`, `i16`, `i32`, `i64`.
- **Sufixos de inteiros sem sinal**: `u8`, `u16`, `u32`, `u64`.
- **Sufixos de float**: `f32`, `f64`.

Sem um sufixo, os literais inteiros assumem o tipo padrão `int` e literais decimais assumem `float`.

São conceitos diferentes:

```ori
const ok: int8 = 12i8

const wrong_type: int8 = 12   // erro: 12 é int, não int8
const wrong_suffix: int = 12i8 // erro: 12i8 é int8, não int
const unknown: i8 = 12        // erro: i8 não é nome de tipo
```

## Coerção & Conversão de Tipos

A Ori não suporta coerção implícita de tipos para evitar bugs ocultos de programação. Em vez disso, as conversões de tipo devem ser feitas explicitamente invocando o nome do tipo primitivo como uma função de conversão:

```ori
const x: float = 3.14
const arredondado: int = int(x) // Converte float para int

const ativo: bool = true
const ativo_str: string = string(ativo) // Converte bool para "true"

const lista_bytes: list<int> = [72, 101, 108, 108, 111]
const bytes_brutos: bytes = bytes(lista_bytes) // Converte listas para bytes brutos
```

## `result` e `optional`

- **`result<T, E>`** — representa sucesso ou erro (tratado via `match` com os braços `success` / `error`).
- **`optional<T>`** — representa presença ou ausência (`some(value)` ou `none`).

As APIs de sistema de arquivos usam `result`, enquanto `io.read_line()` retorna `optional<string>` (retornando `none` ao atingir o fim do arquivo - EOF).

## Condicionais Inline

```ori
const n: int = if flag then 1 else 2
const rotulo: string = if active then "ligado" else "desligado"
```

Ambos os braços da condicional devem ter tipos compatíveis (ou retornar `never`).

## Coleções

`list<T>`, `map<K, V>`, `set<T>`, além de tipos opacos da stdlib (`Deque`, `File`, etc.).

Veja a [referência da stdlib](/reference/) para a documentação completa das APIs.
