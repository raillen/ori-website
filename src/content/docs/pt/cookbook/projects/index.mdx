## Padrão de projeto

Use este formato para projetos Ori pequenos:

1. Coloque tipos de domínio em um módulo.
2. Coloque regras em funções que não imprimem.
3. Coloque I/O de terminal em `main`.
4. Retorne `optional<T>` para dados ausentes.
5. Retorne `result<T, E>` para falhas.
6. Mantenha APIs públicas pequenas.

## Pastas sugeridas

```text
project/
  ori.proj
  main.orl
  app/
    domain.orl
    service.orl
```

Comece com um arquivo se o projeto for pequeno.

Separe quando uma parte tiver responsabilidade clara e nome útil.
