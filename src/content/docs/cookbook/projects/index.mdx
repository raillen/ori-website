## Project standard

Use this shape for small Ori projects:

1. Put domain types in one module.
2. Put rules in functions that do not print.
3. Put terminal I/O in `main`.
4. Return `optional<T>` for missing data.
5. Return `result<T, E>` for failures.
6. Keep public APIs small.

## Suggested folders

```text
project/
  ori.proj
  main.orl
  app/
    domain.orl
    service.orl
```

Start with one file if the project is tiny.

Split when a section has a clear responsibility and a useful name.
