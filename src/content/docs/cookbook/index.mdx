The cookbook is the practical part of the Ori documentation.

Use it when you already know the basic syntax and want examples that show how to write better programs, not just smaller programs.

## What belongs here

- Core language concepts explained with working patterns.
- Tips for names, functions, modules, and error handling.
- Examples that prefer clarity over clever shortcuts.
- Small recipes that can grow into real project code.

## Documentation standard

Every tutorial and example should follow these rules:

1. Start with the reader's goal.
2. Use names that describe the problem domain.
3. Keep functions small and typed at the boundary.
4. Handle `result` and `optional` values explicitly.
5. Avoid showing shortcuts that would be hard to maintain later.
6. Prefer one clear idea per code block.

## Quick checklist

Before publishing an example, check:

- Can a new reader tell what the code is doing?
- Does every binding have a useful name?
- Is the important type information visible?
- Are errors and missing values handled where they appear?
- Is the example short enough to read without jumping around?
